Mount points

There you must specify mount points for your partitions and file system, if not specified before.

Mount points and file systems

Mount point is a partition's place in file ierarchy. There are some common mount points:

  • / is primary mount point. System installs in /, from / system runs.
  • /home mount point for users data. In partition with this mount point stores all users data such as music, video, documents, etc.

It's not all avaliable mount points but the most useful of this.

File system is understanded by computer structure of physical placing files and folders. In AgiliaLinux nowadays supported file systems for installing are:

  • ext2
  • ext3
  • ext4
  • xfs
  • jfs
  • reiserfs
  • btrfs
  • nilfs2

File systems from other OS (FAT16/32, NTFS from Windows, HFS/HFS+ from Mac OS X) supports too but not installable.

File system ext4 recommended for install.

Defining filesystems and mountpoints

You must choose partition, created before, and choose for this file system and mount point. If you format file system earlier, you can choose "Do not format". You must choose one partition with mount point /. Very good practise is choosing another partition for /home. Also, you can create swap-partition for dumping memory if RAM ends. We recommend create swap partition and choose in "Mount Options" "Swap-partition" for this partition.
